--> 자주 발생하는 에러메세지

-- identifier 'V_ENAME' must be declared 뜰 경우 declare 절과 into 절의 변수명이 잘못된 경우 자주 발생.

-- NO DATE FOUND  : SELECT 문에 해당하는 데이터 값이 0이나 없을 경우에 나는 에러. SQL에서는 없어도 오류가 나지는 않으나 PLSQL에서는 오류가 발생함.

-- NUMBER OF ROWS(TO MANY ROWS) : 내가 요구하는 문법상의 답보다 더 많은 내용이 돌아올 경우 발생. SELECT 문에서 정확하게 되어있는지 확인해야함.


-- 치환변수 : plsql을 컴파일 할 때 동작함. declare절과 begin 절 두곳에서 모두 사용 가능하다. 실행되기 전에 대체가 되어야 하기 때문. 실행하면 old 와 new 가 뜨게되며 
old : 기존에 실행되었던 코드.
new :입력한 것을 기반으로 실제 실행되는 코드.
값을 입력할 때 ''안에 값을 넣을 경우 치환변수는 기본적으로 숫자로 인식하기때문에 홀따옴표는 인식되지 않는다. 문자를 넣게되면 인식할 수 없음. 문자를 사용해야할 경우에는 문자 타입을 명시해주거나 '&사원번호'로 만들어줘야한다.


* 기본 sql 문으로 먼저 조회 후 정상처리된다면 그대로 begin절에 가지고 오는 방법 추천
